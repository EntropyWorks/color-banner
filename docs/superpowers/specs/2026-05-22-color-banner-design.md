## Overview

`color-banner` is a standalone Python CLI tool that renders text as colorful figlet ASCII banners in the terminal. It borrows the figlet/pyfiglet rendering concept from the Calligraphy GNOME app but is otherwise a new project — a lightweight, dependency-minimal tool designed for CICD pipelines, shell startup screens, and BBS-style splash screens.

The output can be printed to stdout, saved as a cat-able ANSI file, or exported as a self-contained base64-encoded shell function for embedding in scripts.

## Architecture

### Project Structure

```
color-banner/
├── pyproject.toml           # packaging, entry point, deps (managed with uv)
├── uv.lock
├── LICENSE                  # GPL v3
├── CREDITS.md               # attribution to Calligraphy / GeopJr
├── README.md
├── src/
│   └── color_banner/
│       ├── __init__.py
│       ├── cli.py           # argparse entry point → color_banner command
│       ├── renderer.py      # wraps pyfiglet: text → list[str] of ASCII rows
│       ├── color.py         # hex parsing, palette definitions, gradient math
│       ├── painter.py       # applies color to rendered rows → ANSI escape strings
│       └── output.py        # stdout / --save .ans / --export .sh
└── tests/
    ├── test_color.py
    ├── test_painter.py
    └── test_output.py
```

### Data Flow

```
cli.py  →  renderer.py  →  painter.py  →  output.py
  text        pyfiglet        ANSI           stdout /
  + args      rows            strings        .ans file /
                                             .sh snippet
```

Each module has one responsibility and no circular dependencies. `color.py` is pure math (no I/O) and `painter.py` is a pure transform (rows in, colored rows out) — both are side-effect free and straightforward to unit test.

## Components

### `renderer.py`
Wraps `pyfiglet`. Accepts text and font name, returns `list[str]` — one string per row of the ASCII art. Trailing empty rows are stripped.

### `color.py`
Pure functions, no I/O.

**Built-in palettes** (2–8 hex stop lists):

| Name | Stops |
|------|-------|
| `neon` | `#ff00ff` → `#7700ff` → `#00bbff` → `#00ffdd` |
| `sunset` | `#ff6b35` → `#f7931e` → `#ffd700` → `#ff4e50` |
| `ocean` | `#006994` → `#0099cc` → `#00d4aa` → `#00ff88` |
| `fire` | `#ff0000` → `#ff8800` → `#ffcc00` → `#ffff00` |
| `ice` | `#88d8ff` → `#44aaff` → `#0066ff` → `#0033cc` |
| `rainbow` | `#ff0000` → `#ff8800` → `#ffff00` → `#00cc00` → `#0044ff` → `#8800ff` |

**Gradient math** — normalised `t ∈ [0.0, 1.0]` interpolated across the stop list:

| Direction | `t` formula |
|-----------|-------------|
| `lr` | `col / (max_cols - 1)` |
| `tb` | `row / (num_rows - 1)` |
| `bt` | `1 - row / (num_rows - 1)` |
| `diag` | `(row + col) / (num_rows + max_cols - 2)` |

Linear interpolation between adjacent stops in RGB space. For stop list of length N, the gradient is divided into N-1 equal segments. `gradient_color` returns `tuple[int, int, int]` — a raw RGB triple that `painter.py` formats into the ANSI escape sequence.

### `painter.py`
Takes `rows: list[str]` and a resolved color config (stop list + direction), returns `list[str]` of ANSI-escaped lines.

- Spaces are skipped — no escape code emitted (transparent background preserved)
- Non-space chars are wrapped: `\x1b[38;2;{r};{g};{b}m{char}\x1b[0m`
- Reset appended at end of each line
- When `no_color=True`, returns rows unchanged

### `output.py`
Three output paths:

1. **stdout** — print colored lines directly. If `sys.stdout.isatty()` is `False`, `no_color` is forced automatically (piped output is never garbled).
2. **`--save FILE`** — write raw ANSI string to file with a header comment. `cat FILE` replays the banner.
3. **`--export FILE`** — base64-encode the ANSI output, wrap in a self-contained bash function. Uses `printf '%s'` (not `echo`) to avoid escape/newline interpretation issues. The function name defaults to `show_banner` and is overridable with `--function-name NAME`.

**Export template:**
```bash
#!/usr/bin/env bash
# Generated by color-banner <version>
# Based on Calligraphy by GeopJr (https://codeberg.org/GeopJr/Calligraphy)
# Paste this function into your script or .bashrc
show_banner() {
  printf '%s' "$(printf '%s' '<base64>' | base64 -d)"
}
show_banner
```

## CLI Interface

**Entry point:** `color-banner` (registered in `pyproject.toml` `[project.scripts]`)

```
usage: color-banner TEXT [options]

positional:
  TEXT                    Text to render as ASCII banner

font options:
  -f, --font FONT         Figlet font name (default: slant)
  --list-fonts            Print all available pyfiglet fonts and exit

color options (mutually exclusive):
  --palette NAME          Use a named built-in palette
  --gradient "#HEX ..."   2–8 space-separated hex color stops

  --direction DIR         lr | tb | bt | diag  (default: lr)

output options:
  --save FILE             Write ANSI escape file (.ans)
  --export FILE           Write self-contained shell function (.sh)
  --function-name NAME    Function name for --export (default: show_banner)
  --no-color              Plain text, no ANSI codes

info:
  --list-palettes         Print built-in palette names and hex stops
  -v, --version           Show version and exit
```

If neither `--palette` nor `--gradient` is given, defaults to `--palette neon`.

**Example invocations:**
```bash
# stdout
color-banner "Fox and Dog" --palette neon --direction tb

# save cat-able ANSI file
color-banner "Deploy v2" --palette fire --direction diag --save splash.ans

# export portable shell snippet
color-banner "Deploy v2" --palette fire --direction diag --export splash.sh

# custom hex gradient
color-banner "Hello" --gradient "#ff0080 #7b2fff #00d4ff" --direction lr

# plain figlet text, no color
color-banner "Hello" --font ogre --no-color
```

## Error Handling

| Condition | Message |
|-----------|---------|
| Invalid hex code | `invalid color '#xyz': expected #RRGGBB` |
| Unknown palette | `unknown palette 'foo'. Available: neon, sunset, ocean, fire, ice, rainbow` |
| Unknown font | `unknown font 'bar'. Run --list-fonts to see available fonts` |
| `--save`/`--export` write failure | OS error surfaced as-is, exit code 1 |
| `--gradient` with fewer than 2 stops | `--gradient requires at least 2 color stops` |

All errors go to stderr. Exit code 0 on success, 1 on any error.

## Testing

**`test_color.py`**
- `gradient_color(0.0, stops)` → first stop
- `gradient_color(1.0, stops)` → last stop
- `gradient_color(0.5, ["#000000", "#ffffff"])` → `(127, 127, 127)`
- `parse_hex("#ff0080")` → `(255, 0, 128)`
- `parse_hex("#xyz")` raises `ValueError`

**`test_painter.py`**
- Spaces produce no escape codes
- Non-space chars wrapped with `\x1b[38;2;...m` and `\x1b[0m`
- Each direction produces a distinct color distribution
- `no_color=True` returns plain text

**`test_output.py`**
- `--save` output round-trips: written bytes decode back to original ANSI string
- `--export` output passes `bash -n` syntax check
- Base64 blob in export decodes back to original ANSI string
- Piped stdout (non-TTY) strips ANSI codes automatically

**Runner:** `uv run pytest`
